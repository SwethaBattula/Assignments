🧱 Data Modeling Lab
📌 Overview

This project focuses on data modeling concepts, which form the foundation of any database system.

The objective is to design and structure data in a way that ensures:

✅ Consistency
✅ Efficiency
✅ Scalability

This lab demonstrates how raw data requirements are transformed into a well-structured database schema using proper modeling techniques.

📂 Project Contents
📓 Swetha_Data_Model_Lab.ipynb → Implementation of data modeling concepts
🧠 Concepts Covered
🔹 1. Data Modeling Fundamentals
Understanding entities and attributes
Identifying relationships between entities
Structuring real-world data into tables

📌 Purpose:
Convert business requirements into structured database design

🔹 2. Entity-Relationship (ER) Modeling
Entities (objects like Employee, Department, etc.)
Attributes (properties of entities)
Relationships (connections between entities)

📌 Outcome:
Clear visualization of how data is interconnected

🔹 3. Keys and Constraints
Primary Keys
Foreign Keys
Integrity constraints

📌 Purpose:
Ensure data accuracy and enforce relationships

🔹 4. Normalization
Eliminating redundancy
Improving data consistency
Organizing data into multiple related tables

📌 Normal forms covered:

1NF
2NF
3NF
🔹 5. Schema Design
Designing tables based on relationships
Structuring database for efficient querying
Maintaining logical consistency
